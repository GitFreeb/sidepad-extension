# CLAUDE.md — Проектные инструкции

Инструкции для Claude Code при работе с данным проектом.

## Правила работы

- После каждого изменения логики, архитектуры или поведения расширения — **сразу обновлять этот файл**: добавлять или корректировать соответствующий раздел в «Ключевых архитектурных решениях».
- Не ждать отдельной просьбы от пользователя — обновление инструкций является частью каждой задачи.

## Описание проекта

Chrome-расширение (Manifest V3) — боковая панель с чек-листом задач для повседневного приминения. Открывается как Side Panel при клике на иконку в панели браузера.

## Установка расширения

1. Открыть `chrome://extensions`
2. Включить **Режим разработчика** (переключатель вверху справа)
3. Нажать **Загрузить распакованное расширение** → выбрать папку `Sidepad/`
4. После изменений в JS/HTML/CSS нажать кнопку перезагрузки на карточке расширения

Шаг сборки отсутствует — чистый vanilla JS, без бандлера и зависимостей.

## Архитектура

Все файлы расширения находятся в `Sidepad/`:

- `manifest.json` — Manifest V3; разрешения `sidePanel` и `storage`
- `background.js` — service worker; при установке задаёт `openPanelOnActionClick: true`
- `popup.html` / `popup.css` — интерфейс боковой панели
- `popup.js` — вся логика (состояние, хранилище, рендеринг, импорт)

`Sidepad/default.md` — эталонный формат пустого списка по умолчанию; `DEFAULT_TASKS` в `popup.js` — пустой массив `[]`.

## Ключевые архитектурные решения

**Конвенция ID задач:** встроенные задачи имеют `id ≤ 10000`; пользовательские — `id = Date.now() + 10001` (всегда `> 10000`). Кнопки удаления только для `id > 10000`; «Сбросить выполненные» переводит встроенные задачи в невыполненные, а не удаляет.

**Сохранение состояния:** `chrome.storage.local` хранит `{ tabs, activeTabId, lightMode, lang, autoSave }`. При загрузке восстанавливаются все вкладки, активная вкладка, тема, язык и флаг автосохранения.

**i18n:** константа `I18N` в `popup.js` содержит строки для `ru` и `en`. Дефолтный язык — `en`. Функция `tr(key)` возвращает строку текущего языка. `applyLang(l, save)` обновляет DOM и сохраняет выбор. Переключатель `RU`/`EN` находится в `.tabbar-actions` рядом с кнопкой темы.

**Импорт файлов — `parseMd(content, fileName)`:** авто-определяет формат файла и выбирает парсер:
- **Markdown** (есть строки `- ` или `* `) — заголовки `##` становятся секциями, строки `- `/ `* ` → задачи, `[x]`/`[ ]` → done-флаг. URL извлекаются регулярным выражением.
- **Plain text** (fallback, нет `-`/`*` буллетов) — поддерживает:
  - Нумерованные списки: `1. задача`, `1) задача`
  - Буллеты `*` / `•`
  - Заголовки-секции: строка, за которой `---`/`===` (setext); строка целиком заглавными буквами (≥2 капитальных, ≤60 символов, нет URL); строка, заканчивающаяся на `:` (≤60 символов, нет URL)
  - Признаки выполненной задачи: `[x]`, `✓`, `✔`
  - Обычные строки → задачи в текущую секцию

**Рендеринг:** `render()` полностью перестраивает `innerHTML` при каждом изменении состояния. XSS предотвращается через `escapeHtml()`.

**Секции — имена и приоритет:** внутренний ключ `__default__` удалён. Все секции — обычные строки. Дропдаун показывает плейсхолдер «Выбор секции»/«Select section»; `selectedSection = null` означает ничего не выбрано. При добавлении задачи приоритет секции: 1) выбранная в дропдауне, 2) текст в поле «новая секция», 3) автоимя «Новая секция N»/«New section N» (первый свободный номер). `getNextSectionName(existingSections)` ищет первый свободный номер. `LEGACY_DEFAULT_SECTIONS` — набор `{'__default__', 'Дополнительные задачи', 'Extra tasks'}` для миграции старых данных; `migrateDefaultSection(tab)` конвертирует их при загрузке.

**Автосохранение в Downloads:** управляется кнопкой `↓auto` (`#autoSaveBtn`, `.btn-auto-save`) в `.tabbar-actions`. Состояние `autoSave` (bool, default `true`) хранится в `chrome.storage.local`. `applyAutoSave(on, save)` обновляет кнопку и сохраняет в storage. Загружается в `loadAll()` через `data.autoSave !== false`.

Поведение при **autoSave = ON** (правила по типу вкладки):
- `example` — никогда: ни диалог, ни сохранение.
- `default` — без диалога; сохраняется только если `tasks.length > 0`.
- Любой пустой список — ни диалога, ни сохранения.
- Остальные вкладки с задачами — диалог подтверждения + сохранение.

Поведение при **autoSave = OFF**:
- `example` и `default` — без диалога, без сохранения.
- Импортированные с задачами — диалог «закрыть без сохранения?» → закрыть; без сохранения.
- Пустые — без диалога, без сохранения.

**Кнопка «✕ очистить» — поведение по типу вкладки:**
- `default` — очищает содержимое, таб остаётся; автосохранение если задачи есть; без диалога.
- `example` — закрывает таб и переключается на `default`; без диалога, без сохранения.
- Импортированные вкладки — диалог подтверждения (если задачи есть) → автосохранение → закрывает таб → переключается на `default`.
- Если вкладка `default` не открыта — создаётся автоматически (`switchToOrCreateDefault()`).

**Индикатор скрытых вкладок:** `.tab-strip` обёрнут в `.tab-strip-wrap` (flex: 1, position: relative, overflow: hidden). CSS `::after`/`::before` дают градиентный fade вправо/влево при классах `overflow-right`/`overflow-left`. Два бейджа: `#tabOverflow` (`.tab-overflow-right`) — справа, `#tabOverflowLeft` (`.tab-overflow-left`) — слева. Считаются только полностью невидимые вкладки (частично обрезанные — нет). Правый бейдж: вкладка считается скрытой, если её левый край уже за `visibleEnd - 2`; клик скроллит на 70% вперёд. Левый бейдж: вкладка считается скрытой, если её правый край левее `scrollLeft + 2`; клик делает `scrollTo({ left: 0 })`. `updateTabOverflow()` вызывается из `renderTabs()`, scroll-листенера и ResizeObserver. После перестройки HTML активная вкладка прокручивается в видимую зону через `scrollIntoView`.

**Перемещение задачи в другую секцию:** кнопка `→` (`.btn-move`, `data-action="move-task"`) появляется при наведении на задачу, если существует хотя бы одна другая секция. Клик → `moveMenuTaskId = id` → `render()` генерирует `<ul class="move-section-menu">` с секциями (кроме текущей) прямо в `<li>` задачи, затем позиционирует меню через `position: fixed` и `getBoundingClientRect()` (чтобы оно не обрезалось overflow родителя). Выбор секции вызывает `moveTaskToSection(id, section, 'start')` — задача вставляется в начало целевой секции. Drag-and-drop на заголовок секции вызывает `moveTaskToSection` без параметра (дефолт `'end'`) — вставляет в конец. `Escape` или клик вне меню закрывает его через document click handler. `moveMenuTaskId` сбрасывается при переключении и закрытии вкладки.

**Переименование секции:** кнопка `✎` (`.btn-rename-section`, `data-action="rename-section"`) появляется при наведении на шапку секции. Клик → `renamingSection = name` → `render()` рендерит `<input class="section-rename-input">` вместо `<span class="section-title">`. `Enter`/`blur` → `finishSectionRename()`: обновляет `section` у всех задач и массив `sections`, дедуплицирует. Если новое имя совпадает с существующей секцией — задачи сливаются. `Escape` → отмена. Состояние `renamingSection` сбрасывается при переключении и закрытии вкладки.

**Кнопка «Свернуть / Развернуть»** (`#collapseAllBtn`) в тулбаре: флаг `allCollapsed` (`let allCollapsed = false`). Нажатие «Свернуть» добавляет все секции в `collapsedSections`, флаг → `true`, кнопка меняет текст на «Развернуть». Нажатие «Развернуть» вызывает `resetCollapsedState()` (оставляет свёрнутыми только полностью выполненные секции), флаг → `false`. `updateCollapseBtn()` синхронизирует текст кнопки с флагом. Флаг сбрасывается в `false` при переключении и закрытии вкладки.

**Две функции управления свёрнутым состоянием:**
- `resetCollapsedState()` — полный сброс `collapsedSections`: очищает и заново добавляет только секции, где все задачи выполнены. Вызывается при смене/закрытии вкладки, загрузке, импорте, нажатии «Развернуть».
- `syncAutoCollapse()` — неразрушающая: только *добавляет* секции с выполненными задачами, не трогает ручные свёртки. Вызывается при drag-and-drop, добавлении/удалении задач, чекбоксе. Это гарантирует сохранение ручного состояния при перетаскивании секций.

**Оповещение об обновлениях:** расширение распространяется только как unpacked (без Chrome Web Store), поэтому проверка обновлений реализована вручную в `popup.js`. При каждом открытии панели `loadAll()` вызывает `checkForUpdate()`: сравнивает `chrome.runtime.getManifest().version` с полем `version` из `manifest.json` ветки `main` на GitHub (`MANIFEST_RAW_URL`, числовое сравнение через `compareVersions`). Результат кэшируется в `chrome.storage.local.updateInfo = { lastChecked, latestVersion, dismissedVersion }`, повторный сетевой запрос — не чаще раза в `UPDATE_CHECK_INTERVAL_MS` (24 часа), независимо от успеха/неудачи запроса. Если доступна более новая версия — `renderUpdateBanner()` показывает баннер `#updateBanner` (между tabbar и breadcrumb). Клик по баннеру открывает `REPO_URL` на GitHub; крестик скрывает баннер (`dismissedVersion = latestVersion`) до выхода следующей версии. Бейдж на иконке расширения (`chrome.action.setBadgeText`) не используется — минимальный размер бейджа задаёт сам Chrome, и на маленькой иконке он выглядит слишком грубым; решили оставить только баннер.
