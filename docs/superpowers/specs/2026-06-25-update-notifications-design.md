# Оповещения об обновлениях расширения

## Контекст

Sidepad распространяется только как unpacked-расширение через GitHub (`git@github.com:GitFreeb/sidepad-extension.git`), без Chrome Web Store. Chrome не умеет тихо обновлять такие расширения, поэтому пользователь должен сам узнать о новой версии, подтянуть изменения (`git pull` / скачать заново) и перезагрузить расширение в `chrome://extensions`. Цель фичи — сообщать пользователю, что вышла более новая версия, чем установленная.

## Источник версии

Текущая версия — `chrome.runtime.getManifest().version`.

Актуальная версия репозитория — поле `version` из `manifest.json` ветки `main`, запрос:

```
GET https://raw.githubusercontent.com/GitFreeb/sidepad-extension/main/manifest.json
```

`raw.githubusercontent.com` отдаёт `Access-Control-Allow-Origin: *`, поэтому `fetch` из контекста side panel работает без дополнительных `host_permissions` в `manifest.json`.

**Сравнение версий:** числовое, по сегментам (`"1.10"` считается новее `"1.9"`), не строковое.

## Throttling и хранение состояния

Новый ключ в `chrome.storage.local`:

```js
updateInfo: {
  lastChecked: <timestamp ms>,
  latestVersion: "1.4",     // последняя известная версия из репозитория
  dismissedVersion: "1.3"   // версия, для которой пользователь закрыл баннер крестиком
}
```

Логика проверки выполняется в `popup.js`, вызывается из `loadAll()` при каждом открытии side panel:

1. Если `Date.now() - lastChecked < 24h` — сетевой запрос не делаем, рендерим баннер на основе уже сохранённого `latestVersion` и `dismissedVersion`.
2. Если кэш устарел (или `updateInfo` отсутствует) — делаем `fetch`. Независимо от результата (успех или ошибка) обновляем `lastChecked`, чтобы офлайн-режим не вызывал повторный запрос на каждое открытие панели.
3. При успешном ответе сохраняем `latestVersion` из полученного `manifest.json`.
4. Если `latestVersion` ≤ текущей версии расширения — `updateInfo` сбрасывается в исходное состояние (`latestVersion` = текущая версия, `dismissedVersion` очищается), баннер скрывается.

## UI

**Без бейджа на иконке.** Изначально планировался `chrome.action.setBadgeText`, но минимальный размер «таблетки» бейджа задаёт сам Chrome и визуально он выглядит слишком грубым на маленькой иконке расширения (проверено вручную при тестировании unpacked-сборки) — управлять размером через стандартный API нельзя. От бейджа отказались в пользу только баннера в панели; уведомление видно при открытии side panel, что соответствует частоте использования расширения.

**Баннер в панели** — новый блок в `popup.html` между `.tabbar` и `.breadcrumb`:

```html
<div class="update-banner" id="updateBanner" hidden>
  <span class="update-banner-text" id="updateBannerText"></span>
  <button class="update-banner-close" id="updateBannerClose" title="Скрыть">×</button>
</div>
```

- Текст вида «Доступна версия 1.4 → Обновить» (i18n, ru/en), весь текст — кликабельная ссылка, открывающая `https://github.com/GitFreeb/sidepad-extension` в новой вкладке (`chrome.tabs.create`).
- Баннер показывается, если `latestVersion` > текущей версии **и** `latestVersion !== dismissedVersion`.
- Крестик скрывает баннер и пишет `dismissedVersion = latestVersion` в `chrome.storage.local` — баннер не появится повторно для той же версии, пока не выйдет следующая.

**i18n:** строки баннера и тултипа крестика добавляются в таблицу `I18N` (`ru`/`en`) по тому же паттерну, что остальные строки интерфейса; используются через `tr(key)`.

## Обработка ошибок

- Сеть недоступна, GitHub отдал не-200, `fetch` бросил исключение, JSON не парсится, поле `version` отсутствует/невалидно — ошибка проглатывается (`try/catch`), баннер не показывается, `lastChecked` всё равно обновляется (см. throttling).
- Никаких retry-циклов и алармов — следующая попытка случится при следующем открытии панели, когда кэш протухнет.

## Затронутые файлы

- `popup.js` — функция проверки обновления, вызов из `loadAll()`, рендер/скрытие баннера, обработчики клика/крестика.
- `popup.html` — разметка баннера.
- `popup.css` — стили баннера.
- `manifest.json` — без изменений (новые permissions не требуются).

## Вне рамок (явно не делаем)

- Никакого автоматического скачивания/обновления файлов — только уведомление, обновление вручную пользователем.
- Никаких `chrome.alarms` / фоновых периодических проверок в `background.js` — проверка только при открытии панели.
- Никакого changelog/release notes внутри панели — клик ведёт на GitHub, дальше пользователь сам смотрит историю коммитов.

## Тестирование

Ручное (в проекте нет тестовой инфраструктуры, vanilla JS без сборки):

1. Локально подменить версию в фейковом `manifest.json` (или временно указать fetch на тестовый URL) выше текущей — убедиться, что появляется баннер.
2. Закрыть баннер крестиком — баннер скрылся; переоткрыть панель — баннер не появился повторно для той же версии.
3. Поднять локальную версию в `manifest.json` до уровня `latestVersion` (имитация ручного обновления) — после перезапуска панели баннер исчез, `updateInfo` сброшен.
4. Отключить сеть — открытие панели не виснет, ошибок в консоли нет, поведение как без обновления.
5. Проверить переключение RU/EN — текст баннера меняется вместе с языком.
